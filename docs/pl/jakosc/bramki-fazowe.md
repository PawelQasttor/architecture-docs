# Bramki fazowe

Bramka fazowa to formalny punkt kontrolny między kolejnymi fazami projektu. Przejście do następnej fazy wymaga potwierdzenia, że wszystkie wymagane produkty zostały opracowane, zweryfikowane i zaakceptowane. Poniższe listy kontrolne można wydrukować i stosować bezpośrednio w praktyce projektowej.

::: tip Jak korzystać z list kontrolnych
Przed bramką kierownik projektu przechodzi przez wszystkie punkty. Każdy punkt oznaczony `[ ]` wymaga potwierdzenia. Brak spełnienia któregokolwiek punktu oznacza wstrzymanie przejścia do kolejnej fazy do czasu uzupełnienia.
:::

Szczegóły poszczególnych faz znajdziesz w [opisach faz projektu](/pl/fazy/rozpoczecie). Zasady prowadzenia przeglądów opisano w [procedurach przeglądów](./procedury-przegladow).

---

## Bramka 1 -- Rozpoczęcie → Koncepcja

**Odpowiedzialny:** kierownik projektu

- [ ] Karta Budynku utworzona z danymi identyfikacyjnymi (adres, działka, inwestor)
- [ ] Warunki zabudowy lub wypis z MPZP dołączony i zweryfikowany
- [ ] Mapa do celów projektowych aktualna (nie starsza niż 6 miesięcy)
- [ ] Wytyczne inwestora udokumentowane i potwierdzone
- [ ] Program funkcjonalno-użytkowy uzgodniony
- [ ] Zespół projektowy powołany, role przypisane ([zarządzanie projektem](/pl/zarzadzanie-projektem/zarzadzanie))
- [ ] Harmonogram projektu ustalony z terminami bramek
- [ ] **BIM:** [Plan Realizacji BIM (BEP)](/pl/integracja-bim/bep) opracowany, poziom LOD 100 zdefiniowany
- [ ] **Przepisy:** wstępna identyfikacja wymagań z [Warunków Technicznych](/pl/przepisy/) i przepisów pożarowych

---

## Bramka 2 -- Koncepcja → Projekt wstępny

**Odpowiedzialny:** główny projektant

- [ ] Warianty koncepcyjne opracowane (minimum 2) i przedstawione inwestorowi
- [ ] Wariant wybrany -- decyzja inwestora udokumentowana
- [ ] Karty Stref utworzone dla wszystkich stref funkcjonalnych
- [ ] Wstępny bilans powierzchni sporządzony (PUM, PUK, powierzchnia całkowita)
- [ ] Schemat funkcjonalny budynku opracowany
- [ ] Wstępna analiza nasłonecznienia i zacienienia wykonana
- [ ] Przegląd koleżeński koncepcji przeprowadzony -- [procedura przeglądu](./procedury-przegladow)
- [ ] Protokół z przeglądu z inwestorem podpisany
- [ ] **BIM:** model koncepcyjny 3D zgodny z LOD 200
- [ ] **Przepisy:** potwierdzenie zgodności z MPZP / warunkami zabudowy (linie zabudowy, wysokość, intensywność)

---

## Bramka 3 -- Projekt wstępny → Projekt budowlany

**Odpowiedzialny:** główny projektant + branżyści

- [ ] Karty Pomieszczeń utworzone dla wszystkich pomieszczeń
- [ ] Karty Kondygnacji kompletne z oznaczeniami osi i poziomów
- [ ] Karty Instalacji wstępnie opracowane (wod.-kan., HVAC, elektro)
- [ ] Przegląd koordynacyjny międzybranżowy przeprowadzony -- brak krytycznych kolizji
- [ ] Bilans powierzchni zaktualizowany i zgodny z programem
- [ ] Koncepcja konstrukcji uzgodniona z konstruktorem
- [ ] Karty Wymagań pożarowych i akustycznych wypełnione
- [ ] Wstępna opinia geologiczna uzyskana (jeśli wymagana)
- [ ] Protokół przeglądu z inwestorem podpisany
- [ ] **BIM:** model wielobranżowy LOD 300, [kontrola kolizji](/pl/integracja-bim/) wykonana
- [ ] **Przepisy:** weryfikacja [zgodności z WT](/pl/przepisy/) -- drogi ewakuacyjne, dostępność, nasłonecznienie

---

## Bramka 4 -- Projekt budowlany → Projekt wykonawczy

**Odpowiedzialny:** główny projektant

- [ ] Projekt zagospodarowania terenu kompletny
- [ ] Rzuty, przekroje, elewacje w wersji do pozwolenia na budowę
- [ ] Opis techniczny budowlany opracowany
- [ ] Wszystkie branże złożone (architektura, konstrukcja, instalacje sanitarne, elektryczne)
- [ ] Karty Wymagań zweryfikowane pod kątem kompletności
- [ ] Uzgodnienia z gestorami mediów uzyskane (woda, gaz, energia, telekomunikacja)
- [ ] Uzgodnienie z rzeczoznawcą ds. ppoż. uzyskane
- [ ] Uzgodnienie z rzeczoznawcą ds. higieniczno-sanitarnych (jeśli wymagane)
- [ ] [Kontrola dokumentów](/pl/zarzadzanie-projektem/kontrola-dokumentow) -- wszystkie pliki z poprawnym statusem i wersją
- [ ] Przegląd kompletności dokumentacji do wniosku o PnB
- [ ] **BIM:** model LOD 300 zaktualizowany, eksport do formatu IFC
- [ ] **Przepisy:** pełna [weryfikacja zgodności](/pl/przepisy/) -- WT, przepisy pożarowe, akustyka, energetyka

::: warning Projekt budowlany to etap formalny
Dokumentacja projektu budowlanego jest składana do urzędu. Błędy na tym etapie skutkują odmową wydania pozwolenia na budowę lub koniecznością uzupełnień, co opóźnia cały projekt.
:::

---

## Bramka 5 -- Projekt wykonawczy → Budowa

**Odpowiedzialny:** główny projektant + kierownik projektu

- [ ] Detale architektoniczne opracowane (węzły, połączenia, obróbki)
- [ ] Karty Urządzeń kompletne z parametrami technicznymi
- [ ] Specyfikacje materiałowe sporządzone
- [ ] Przedmiary robót opracowane dla wszystkich branż
- [ ] Koordynacja międzybranżowa zakończona -- brak kolizji
- [ ] Zestawienia stolarki, ślusarki, wykończenia opracowane
- [ ] Schematy instalacji wykonawcze kompletne
- [ ] Plan bezpieczeństwa i ochrony zdrowia (BIOZ) opracowany
- [ ] Przegląd koleżeński dokumentacji wykonawczej przeprowadzony
- [ ] Protokół przekazania dokumentacji wykonawczej wykonawcy
- [ ] **BIM:** model LOD 350/400, koordynacja zakończona, [raport kolizji](/pl/integracja-bim/) czysty
- [ ] **Przepisy:** weryfikacja detali pod kątem norm (np. PN-EN, normy akustyczne, termiczne)

---

## Bramka 6 -- Budowa → Dokumentacja powykonawcza

**Odpowiedzialny:** kierownik budowy + inspektor nadzoru + projektant (nadzór autorski)

- [ ] Dziennik budowy prowadzony bez przerw
- [ ] Protokoły nadzorów autorskich kompletne
- [ ] Zmiany projektowe wprowadzone w trakcie budowy udokumentowane
- [ ] Karty zmian opisane z uzasadnieniem i akceptacją projektanta
- [ ] Protokoły odbiorów częściowych (roboty zanikające) kompletne
- [ ] Badania i pomiary wykonane (szczelność, rezystancja, wydajność wentylacji)
- [ ] **BIM:** model aktualizowany o zmiany budowlane (as-built), LOD 500
- [ ] **Przepisy:** odstępstwa od projektu ocenione pod kątem zgodności z pozwoleniem na budowę

---

## Bramka 7 -- Dokumentacja powykonawcza → Przekazanie

**Odpowiedzialny:** kierownik projektu

- [ ] Dokumentacja powykonawcza opracowana (rysunki powykonawcze wszystkich branż)
- [ ] Karty Pomieszczeń zaktualizowane o stan faktyczny
- [ ] Karty Urządzeń uzupełnione o dane z realizacji (numery seryjne, parametry rzeczywiste)
- [ ] Karty Instalacji zaktualizowane -- trasy, średnice, nastawy
- [ ] Protokoły prób i badań kompletne
- [ ] Świadectwo charakterystyki energetycznej sporządzone
- [ ] Inwentaryzacja geodezyjna powykonawcza wykonana
- [ ] **BIM:** model as-built LOD 500 zweryfikowany z pomiarami
- [ ] **Przepisy:** potwierdzenie zgodności realizacji z pozwoleniem na budowę i projektem

---

## Bramka 8 -- Przekazanie (zamknięcie projektu)

**Odpowiedzialny:** kierownik projektu + inwestor

- [ ] Dokumentacja użytkowa budynku przekazana (instrukcje obsługi, schematy)
- [ ] Instrukcje eksploatacji instalacji opracowane
- [ ] Książka obiektu budowlanego założona
- [ ] Harmonogram przeglądów okresowych sporządzony
- [ ] Szkolenie użytkownika / zarządcy przeprowadzone
- [ ] Protokół przekazania obiektu podpisany
- [ ] Gwarancje i rękojmie skatalogowane
- [ ] Archiwizacja dokumentacji projektowej zakończona ([kontrola dokumentów](/pl/zarzadzanie-projektem/kontrola-dokumentow))
- [ ] **BIM:** model przekazany w formacie IFC i natywnym, zgodnie z [BEP](/pl/integracja-bim/bep)
- [ ] **Przepisy:** zawiadomienie o zakończeniu budowy / pozwolenie na użytkowanie uzyskane

---

## Podsumowanie wymagań BIM na bramkach

| Bramka | Poziom LOD | Zakres modelu | Format wymiany |
|--------|-----------|---------------|----------------|
| 1 | LOD 100 | Bryła, otoczenie | -- |
| 2 | LOD 200 | Strefy, układ funkcjonalny | IFC (opcjonalnie) |
| 3 | LOD 300 | Wielobranżowy, elementy budowlane | IFC |
| 4 | LOD 300 | Dokumentacja do PnB | IFC |
| 5 | LOD 350/400 | Detale, instalacje | IFC + natywny |
| 6 | LOD 500 | Zmiany budowlane | IFC + natywny |
| 7 | LOD 500 | As-built | IFC + natywny |
| 8 | LOD 500 | Model przekazany | IFC + natywny |

::: info Szczegóły integracji BIM
Wymagania dotyczące modelu BIM na poszczególnych etapach są opisane w sekcji [Integracja BIM](/pl/integracja-bim/). Plan Realizacji BIM (BEP) definiuje szczegółowe zasady dla każdego projektu -- zobacz [BEP](/pl/integracja-bim/bep).
:::
