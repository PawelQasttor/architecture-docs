# Telemetry stream — CO₂ in Bedroom 4.02 (TEL-CO2-402-001)

A `telemetry_stream` entity (**new in SBM v2.2**) wrapping the BMS-recorded
CO₂ readings from apartment 4.02 Bedroom 01. Replaces the narrative
"reference to external CSV" pattern used in the v2.0 version of this
example with a first-class entity the compiler can reason about.

The design-target breach drove [`ISS-ANOMALY-CO2-001`](../issues/ISS-ANOMALY-CO2-001)
and the subsequent retro-commissioning rebalance under
[`ISS-RETROCX-MVHR-001`](../issues/ISS-RETROCX-MVHR-001).

## Why this entity matters in v2.2

In SBM v2.0, this information lived as narrative + a stale `dataSource`
string reference. The compiler had no way to:
- **Validate** that the threshold breach was correctly flagged
- **Detect** new breaches automatically
- **Cross-reference** stream → issue → retrocx-action chains

With the v2.2 `telemetry_stream` entity, every measurement that drives
operation-phase decisions becomes a first-class queryable record.
