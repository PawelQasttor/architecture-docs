## Key product decisions made along the way

| Decision | Choice |
|---|---|
| Canonical SBM expansion | **"Semantic Building Model"** (matched the 23 user-facing docs; fixed the 4 places that said "Structured") |
| Phase-model mapping | initiation→`concept`, schematic→`schematic_design`, as-built→`commissioning`, handover→`operation`; new: `bidding_procurement`, `renovation`, `decommissioned` |
| Phase tool fidelity (A1 scope) | **Full rewrite** of 10 pages EN + PL (≈7,400 lines) vs. minimal — full was chosen |
| Compiler back-compat for numeric `--phase` | **Kept** via `LEGACY_PHASE_MAP` so existing tests/fixtures pass unchanged |
| `cost.basis` for the stairwell void | `"override"` + explanation (rather than `"not_applicable"` which isn't in the space-cost enum) |
| Report i18n strategy | Stable `code`s in JSON + render-time localization in `html-report.mjs` (machine output stays English/stable; presentation is localized) |
| Schema vocabulary in PL docs | Identifier/enum values (`entityType:"building"`, `status:"compliant"`, phase/confidence IDs) **stay identical EN/PL** by design |
| Diagrams scope | LOD-100 schematic from current data; **no `geometry:` schema extension** yet — that would be a separate decision |
| Releasing tag | Re-tagged `v2.1.0` while it was still unpublished (safe); after publishing, post-release work goes to `[Unreleased]` |
