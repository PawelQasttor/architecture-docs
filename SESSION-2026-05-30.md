## Versioning note (the tooling-ahead-of-spec offset)
- **Spec** = `sbm_version` + schema file (currently **2.5**, `sbm-schema-v2.5.json`).
- **Tooling** = `package.json` + compiler `VERSION` (currently **2.7.0**).
- The offset began with the v2.5.0 tooling-only graph release (spec stayed
  2.4) and mirrors the earlier v2.1.0 precedent. Each is source of truth for
  its own layer.

## Conventions reinforced this session
- Adding an entity type = ~12 touch points (schema $def + bucket + sbm_version;
  constants V2_x set + ALL; parse validTypes; normalize 4 anchors; validate
  schema-path + rules; quality profile; index VERSION + sbm_version; targets;
  tests; CHANGELOG; CLAUDE.md; nav config.ts both locales; examples PL+EN;
  extract:json).
- `responsibility.discipline` is an enum (`architectural`, not "architecture";
  no "general"/"environmental"/"facilities"). YAML `field: null` fails schema ‚Äî
  omit instead.
- `npm run extract:json` re-stamps `extractedAt` on all ~249 JSON exports;
  stage only real-delta files (new entity exports + the 4 root aggregates) and
  `git restore --worktree docs/public/json` to discard timestamp churn. Tooling
  releases that only touch `scripts/` + `build/` (gitignored) produce no churn.
